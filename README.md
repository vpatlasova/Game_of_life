# Игра "Жизнь" 

## Описание задания

Реализовать известный клеточный автомат под названием игра "жизнь". Используйте тороидальную геометрию.

### Дополнительные задания:

1. Реализуйте на том же движке игру по альтернативными правилам.
2. Реализуйте возможность добавления известных фигур на поле перетаскиванием.

## Достигнутая сложность и реализация

### 1. Выполненные основные требования:

- Реализован клеточный автомат «Игра жизнь»: построена полная логика обновления поколений по правилам Конвея
- Использована тороидальная геометрия: поле зациклено по горизонтали и вертикали.
- Код соответствует требованиям ООП: созданы классы Field и Panel
- Настройки вынесены в конфигурационный файл: используется config.json для размеров поля, скорости, цветов, правил игры и списка фигур
- Реализована корректная анимация через pygame

### 2. Дополнительная функциональность:

- Реализована поддержка альтернативных правил: в конфигурационном файле добавлен выбор одного из восьми наборов правил, автоматически применяемых при запуске программы (Conway, HighLife, Seeds, LifeWithoutDeath, Replicator, DayAndNight, Diamoeba, Anneal)
- Реализовано добавление известных фигур на поле перетаскиванием: в конфигурационный файл добавлены готовые паттерны (block, blinker, toad, pulsar, glider, lwss, mwss, hwss), каждая фигура может быть выбрана пользователем в панели справа и размещена на поле методом drag-and-drop.

### 3. Оригинальные расширения:

- Регулировка скорости симуляции с клавиатуры клавишами A/Z: пользователь может динамически увеличивать или уменьшать задержку между поколениями без перезапуска программы.
- Автоматическая остановка симуляции при достижении стабильной конфигурации: если новое поколение полностью совпадает с предыдущим, игра прекращает работу.
- Выбор первоначального состояния поля загружается из текстовых файлов field.txt или field1.txt
